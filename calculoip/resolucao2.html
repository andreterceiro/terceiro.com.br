Resolução do exercício 2


      Primeiro vamos verificar em que classe está o endereço. Convertendo o primeiro octeto para binário, temos:

200 (10) = 110 00000 (2)


      Descobrimos então que o endereço pertence à classe C, ou seja, podemos utilizar apenas o último octeto para dividir em sub-redes. Para dividir o endereço em 10 sub-redes utilizáveis devmos pegar emprestado 4 bits do campo host, pois 24-2 = 14. Lembrando que subtraimos 2 porque não utilizamos a primeira nem a última sub-rede. Como pegamos 2 bits, o campo host está menor. Antes ele possuia 8 bits, agora possui 4 (8 - 4 = 4). Temos que verificar se 4 bits atendem nossa necessidade para o número de hosts, então calculamos:

24-2=14


      Lembrando que subtraimos 2 agora porque o primeiro endereço de cada sub-rede representa a própria sub-rede e o último é utilizado para broadcast dentro da sub-rede. Nossa necessidade é de 14 hosts por sub-rede, então temos nossa necessidade satisfeita.       Vamos então calcular a máscara de sub-rede. Tínhamos antes do cálculo 24 bits para rede e 8 para host. Após nosso cálculo, temos 24 bits para rede, 4 para sub-rede e 4 para host. Como tanto os bits de rede e sub-rede são representados pelo algarismo "1" (em binário) e na prática representam a mesma coisa, poderíamos dizer também que temos 28 bits para rede/sub-rede e 4 para hosts. A máscara fica então assim:

11111111.1111111.11111111.11110000

* 1 = rede/sub-rede , 0 = host


      Convertendo para decimal, temos:

255.255.255.240


      Lembrando que quando convertemos a máscara de binário para decimal, "esquecemos" a divisão rede/sub-rede - host e analisamos apenas a divisão por octetos.

      Agora vamos definir qual é o endereço da última sub-rede utilizável. Como não alteramos os três primeiros octetos, que representam a rede, não precisamos analisá-lo. Vamos analisar apenas o último octeto. A última sub-rede tem como valor no último octeto:

11110000

4 bits de sub-rede, 4 bits de host


      Três detalhes muito importantes:
  • O campo host está preenchido todo com o algarismo 0. É assim que ele deve estar quando calculamos um endereço de sub-rede
  • A última sub-rede tem o valor 1111 em binário, que é igual a 15 em decimal;
  • O campo sub-rede está preenchido com o valor da última sub-rede, mas ela não é utilizável (a primeira e a última sub-redes não são utilizáveis).

      Já que a última sub-rede (sub-rede 0) não é utilizável, vamos calcular o endereço da primeira sub-rede utilizável. Em binário, teremos este valor no último octeto:

11100000

      4 bits de sub-rede, 4 bits de host


      Analisando novamente alguns detalhes muito importantes:
  • O campo host está preenchido todo com o algarismo 0. É assim que ele deve estar quando calculamos um endereço de sub-rede
  • A última sub-rede utilizável tem o valor 1110 em binário, que é igual a 14 em decimal. Lembre-se que calculamos que teríamos 14 sub-redes utilizáveis (24-2=14). Então a última sub-rede é a sub-rede 14;

      Descobrimos o valor em binário, convertendo para decimal, temos:

11100000(2) = 224(10)


      Então vamos escrever o endereço completo, "anexando" o octeto que calculamos aos três primeiros octetos:

200.214.150.224


      Agora devemos calcular o endereço de broadcast da última sub-rede utilizável. Voltaremos para o número em binário e iremos manter o valor dos bits que representam a sub-rede, trocando apenas o valor dos bits do campo host de 0 para 1, pois para termos um broadcast dentro da sub-rede, todos os bits do campo host devem ser preenchidos com o valor 1. Veja:

11101111

      4 bits de sub-rede, 4 bits de host - todos preenchidos com 1 (broadcast)


      Descobrimos o valor em binário, convertendo para decimal, temos:

11101111(2) = 239(10)


      Então vamos escrever o endereço completo, "anexando" o octeto que calculamos aos três primeiros octetos:

200.214.150.239


Ok..., agora falta apenas classificar o endereço 200.214.150.143. Vamos analisar o último octeto, começando convertendo o valor dele:

143(10) = 10001111(2)


      Vamos analisar na estrutura sub-rede / host:

10001111


      Analisando alguns detalhes muito importantes:
  • O campo host não está preenchido todo com o algarismo 0, então este endereço não representa uma sub-rede.
  • O campo host está preenchido todo com o algarismo 1, então este endereço representa um broadcast.

    Para descobrirmos a qual sub-rede pertence este endereço de broadcast, podemos utilizar dois métodos:
  1. Colocar o valor 0 em todos os bits de host, sem alterar os bits de rede e sub-rede.
  2. Aplicar uma operação AND do endereço com a máscara de sub-rede

      Com os dois métodos obtemos o mesmo resultado. Eu prefiro o 1º método. Vamos analisar apenas o último octeto, pois neste caso não alteramos o 1º, o 2º nem o 3º octetos. O valor dele é 10001111 (143 em binário). O valor dos bits de sub-rede é 1000 e o valor dos bits do campo host é 1111. Conforme descrevemos anteriormente, vamos alterar o valor dos bits do campo host para 0 e obteremos assim a sub-rede a qual o host pertence. O octeto em binário ficará assim:

10000000.


      Convertendo o resultado obtido para decimal e anexando-o aos três outros octetos, obteremos a resposta:

10000000(2) = 128(10)


      Concluindo, o host 200.214.150.143, utilizando a máscara 255.255.255.240, pertence a sub-rede 200.214.150.128.


      Vamos demonstrar também como obter o mesmo resultado através de uma operação AND com a máscara de sub-rede. Primeiro vamos passar o endereço 200.214.150.143 e a máscara 255.255.255.240 para binário:

Decimal Binário
200.214.150.143 11001000.11010110.10010110.10001111
255.255.255.240 11111111.11111111.11111111.11110000


      Vamos agora aplicar uma operação AND, algarimo por algarismo. Caso tenha dúvida em como efetuar esta operação, favor visite a seção operações lógicas

11001000.11010110.10010110.10001111
11111111.11111111.11111111.11110000
---------------------------------------------
11001000.11010110.10010110.10000000


      Convertendo 11001000.11010110.10010110.10000000 para decimal, obtemos 200.214.150.128, o mesmo resultado obtido com o outro método.

    Finalizando, vamos escrever os resultados obtidos:
  • Máscara = 255.255.255.240
  • Endereço da última sub-rede utilizável = 200.214.150.224
  • Endereço de broadcast da última sub-rede utilizável = 200.214.150.239
  • Classificação do endereço 200.214.150.143 = broadcast
  • A sub-rede a qual pertence o broadcast 200.214.150.143 é 200.214.150.128
