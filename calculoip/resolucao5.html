Resolução do exercício 5


      Primeiro vamos verificar em que classe está o endereço. Convertendo o primeiro octeto para binário, temos:

221 (10) = 110 11101 (2)


      Descobrimos então que o endereço pertence é classe C, ou seja, podemos utilizar apenas o último octeto para dividir em sub-redes. Para dividir o endereço em 8 sub-redes devmos pegar emprestado 3 bits do campo host, pois 23= 8. Veja que o exercício informou que devemos calcular 8 sub-redes, independente de serem utilizáveis. Sendo assim, não subtraimos 2 do resultado.
      Como pegamos 3 bits emprestados, o campo host está menor. Antes ele possuia 8 bits, agora possui 5 (8 - 3= 5). Temos que verificar se 5 bits atendem nossa necessidade para o número de hosts, então calculamos:

25-2=30


      Lembrando que subtraimos 2 agora porque o primeiro endereço de cada sub-rede representa a própria sub-rede e o último é utilizado para broadcast dentro da sub-rede. Nossa necessidade é de 30 hosts por sub-rede, então temos nossa necessidade satisfeita.
      Vamos então calcular a máscara de sub-rede. Tínhamos antes do cálculo 24 bits para rede e 8 para host. Após nosso cálculo, temos 24 bits para rede, 3 para sub-rede e 5 para host. Como tanto os bits de rede e sub-rede são representados pelo algarismo "1" (em binário) e na prática representam a mesma coisa, poderíamos dizer também que temos 27 bits para rede/sub-rede e 5 para hosts. A máscara fica então assim:

11111111.1111111.11111111.11100000

* 1= rede/sub-rede , 0= host


      Convertendo para decimal, temos:

255.255.255.224


      Lembrando que quando convertemos a máscara de binário para decimal, "esquecemos" a divisão rede/sub-rede - host e analisamos apenas a divisão por octetos.


      Vamos agora classificar o endereço 200.10.15.50. Vamos analisar o último octeto, começando convertendo o valor dele:

50(10)= 00110010(2)


      Vamos analisar na estrutura sub-rede / host:

00110010


      Analisando alguns detalhes muito importantes:
  • O campo host não está preenchido todo com o algarismo 0, então este endereço não representa uma sub-rede.
  • O campo host não está preenchido todo com o algarismo 1, então este endereço não representa um broadcast.
  • Não sendo uma sub-rede ou um broadcast, ele é um endereço de host

      Para descobrirmos a qual sub-rede pertence este host, podemos utilizar dois métodos:
  1. Colocar o valor 0 em todos os bits de host, sem alterar os bits de rede e sub-rede.
  2. Aplicar uma operacão AND do endereço com a máscara de sub-rede

     Com os dois métodos obtemos o mesmo resultado. Eu prefiro o 1o método. Vamos analisar apenas o último octeto, pois neste caso não alteramos o 1o, o 2o nem o 3o octetos. O valor dele é 00110010 (50 em binário). O valor dos bits de sub-rede é 001 e o valor dos bits do campo host é 10010. Conforme descrevemos anteriormente, vamos alterar o valor dos bits do campo host para 0 e obteremos assim a sub-rede a qual o host pertence. O octeto em binário ficará assim:

00100000.


      Convertendo o resultado obtido para decimal e anexando-o aos três outros octetos, obteremos a resposta:

00100000(2)= 32(10)


      Concluindo, o host 221.10.15.50, utilizando a máscara 255.255.255.224, pertence a sub-rede 221.10.15.32.


      Vamos demonstrar também como obter o mesmo resultado através de uma operacão AND com a máscara de sub-rede. Primeiro vamos passar o endereço 221.10.15.50 e a máscara 255.255.255.224 para binário:

      Vamos agora aplicar uma operacão AND, algarimo por algarismo. Caso tenha dúvida em como efetuar esta operacão, favor visite a serão operações lógicas

11011101.00001010.00001111.00100000


      Convertendo 11011101.00001010.00001111.00100000 para decimal, obtemos 221.10.15.32, o mesmo resultado obtido com o outro método.
