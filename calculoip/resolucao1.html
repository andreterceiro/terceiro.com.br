Resolução do exercício 1


      Primeiro vamos verificar em que classe está o endereço. Convertendo o primeiro octeto para binário, temos:

192 (10) = 110 00000 (2)


      Descobrimos então que o endereço pertence à classe C, ou seja, podemos utilizar apenas o último octeto para dividir em sub-redes. Para dividir o endereço em 2 sub-redes utilizáveis devmos pegar emprestado 2 bits do campo host, pois 22-2 = 2. Lembrando que subtraimos 2 porque não utilizamos a primeira nem a última sub-rede. Como pegamos 2 bits, o campo host está menor. Antes ele possuia 8 bits, agora possui 6 (8 - 2 = 6). Temos que verificar se 6 bits atendem nossa necessidade para o número de hosts, então calculamos:

26-2=62


      Lembrando que subtraimos 2 agora porque o primeiro endereço de cada sub-rede representa a própria sub-rede e o último é utilizado para broadcast dentro da sub-rede. Nossa necessidade é de 60 hosts por sub-rede, então com 62 hosts temos nossa necessidade satisfeita.       Vamos então calcular a máscara de sub-rede. Tínhamos antes do cálculo 24 bits para rede e 8 para host. Após nosso cálculo, temos 24 bits para rede, 2 para sub-rede e 6 para host. Como tanto os bits de rede e sub-rede são representados pelo algarismo "1" (em binário) e na prática representam a mesma coisa, poderíamos dizer também que temos 26 bits para rede/sub-rede e 6 para hosts. A máscara fica então assim:


11111111.1111111.11111111.11000000

* 1 = rede/sub-rede , 0 = host


      Convertendo para decimal, temos:

255.255.255.192



      Lembrando que quando convertemos a máscara de binário para decimal, "esquecemos" a divisão rede/sub-rede - host e analisamos apenas a divisão por octetos.

      Agora vamos definir qual é o endereço da primeira sub-rede utilizável. Como não alteramos os três primeiros octetos, que representam a rede, não precisamos analisá-lo. Vamos analisar apenas o último octeto. A primeira sub-rede (chamada normalmente de sub-rede 0) tem como valor no último octeto:

00000000

      2 bits de sub-rede, 6 bits de host


      Três detalhes muito importantes:
  • O campo host está preenchido todo com o algarismo 0. É assim que ele deve estar quando calculamos um endereço de sub-rede
  • A sub-rede 0 tem o valor 00 em binário, que é igual a 0 em decimal;
  • O campo sub-rede está preenchido com o valor da primeira sub-rede, mas ela não é utilizável (a primeira e a última sub-redes não são utilizáveis).
      Já que a primeira sub-rede (sub-rede 0) não é utilizável, vamos calcular o endereço da primeira sub-rede utilizável. Em binário, teremos este valor no último octeto:

01000000

      2 bits de sub-rede, 6 bits de host


      Analisando novamente alguns detalhes muito importantes:
  • O campo host está preenchido todo com o algarismo 0. É assim que ele deve estar quando calculamos um endereço de sub-rede
  • A sub-rede 1 tem o valor 01 em binário, que é igual a 1 em decimal;

      Descobrimos o valor em binário, convertendo para decimal, temos:

01000000(2) = 64(10)


      Então vamos escrever o endereço completo, "anexando" o octeto que calculamos aos três primeiros octetos:

192.168.0.64


      Agora devemos calcular o endereço de broadcast da 1ª sub-rede utilizável. Voltaremos para o número em binário e iremos manter o valor dos bits que representam a sub-rede, trocando apenas o valor dos bits do campo host de 0 para 1, pois para termos um broadcast dentro da sub-rede, todos os bits do campo host devem ser preenchidos com o valor 1. Veja:

01111111
      2 bits de sub-rede, 6 bits de host - todos preenchidos com 1 (broadcast)


      Descobrimos o valor em binário, convertendo para decimal, temos:

01111111(2) = 127(10)


      Então vamos escrever o endereço completo, "anexando" o octeto que calculamos aos três primeiros octetos:

192.168.0.127


      Ok..., agora falta apenas classificar o endereço 192.168.0.140. Vamos analisar o último octeto, começando convertendo o valor dele:

140(10) = 10001100(2)


      Vamos analisar na estrutura sub-rede / host:

10001100


      Analisando alguns detalhes muito importantes:
  • O campo host não está preenchido todo com o algarismo 0, então este endereço não representa uma sub-rede.
  • O campo host não está preenchido todo com o algarismo 1, então este endereço não representa um broadcast.
  • Não sendo uma sub-rede ou um broadcast, ele é um endereço de host

    Para descobrirmos a qual sub-rede pertence este host, podemos utilizar dois métodos:
  1. Colocar o valor 0 em todos os bits de host, sem alterar os bits de rede e sub-rede.
  2. Aplicar uma operação AND do endereço com a máscara de sub-rede

      Com os dois métodos obtemos o mesmo resultado. Eu prefiro o 1º método. Vamos analisar apenas o último octeto, pois neste caso não alteramos o 1º, o 2º nem o 3º octetos. O valor dele é 10001100 (140 em binário). O valor dos bits de sub-rede é 10 e o valor dos bits do campo host é 001100. Conforme descrevemos anteriormente, vamos alterar o valor dos bits do campo host para 0 e obteremos assim a sub-rede a qual o host pertence. O octeto em binário ficará assim:

10000000.


      Convertendo o resultado obtido para decimal e anexando-o aos três outros octetos, obteremos a resposta:

10000000(2) = 128(10)


      Concluindo, o host 192.168.0.140, utilizando a máscara 255.255.255.192, pertence a sub-rede 192.168.0.128.


      Vamos demonstrar também como obter o mesmo resultado através de uma operação AND com a máscara de sub-rede. Primeiro vamos passar o endereço 192.168.0.140 e a máscara 255.255.255.192 para binário:

Decimal Binário
192.168.0.140 11000000.10101000.00000000.10001100
255.255.255.192 11111111.11111111.11111111.11000000


      Vamos agora aplicar uma operação AND, algarimo por algarismo. Caso tenha dúvida em como efetuar esta operação, favor visite a seção operações lógicas

11000000.10101000.00000000.10001100
11111111.11111111.11111111.11000000
---------------------------------------------
11000000.10101000.00000000.10000000


      Convertendo 11000000.10101000.00000000.10000000 para decimal, obtemos 192.168.0.128, o mesmo resultado obtido com o outro método.

    Finalizando, vamos escrever os resultados obtidos:
  • Máscara = 255.255.255.192
  • Endereço da primeira sub-rede utilizável = 192.168.0.64
  • Endereço de broadcast da 1ª sub-rede utilizável = 192.168.0.127
  • Classificação do endereço 192.168.0.140 = host
  • A sub-rede a qual pertence o host 192.168.0.140 é 192.168.0.128
