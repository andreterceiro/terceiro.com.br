Resolução do exercício 3


      Primeiro vamos verificar em que classe está o endereço. Convertendo o primeiro octeto para binário, temos:

140 (10) = 10 001100 (2)


      Descobrimos então que o endereço pertence à classe B, ou seja, podemos utilizar apenas os dois últimos octetos para dividir em sub-redes. Para dividir o endereço em 100 sub-redes utilizáveis devmos pegar emprestado 7 bits do campo host, pois 27-2 = 126. Lembrando que subtraimos 2 porque não utilizamos a primeira nem a última sub-rede. Como pegamos 7 bits, o campo host está menor. Antes ele possuia 16 bits (somando os dois octetos), agora possui 9 (16 - 7 = 9). Temos que verificar se 9 bits atendem nossa necessidade para o número de hosts, então calculamos:

29-2=510.


      Lembrando que subtraimos 2 agora porque o primeiro endereço de cada sub-rede representa a própria sub-rede e o último é utilizado para broadcast dentro da sub-rede. Nossa necessidade é de 500 hosts por sub-rede, então temos nossa necessidade satisfeita.
      Vamos então calcular a máscara de sub-rede. Tínhamos antes do cálculo 16 bits para rede e 16 para host. Após nosso cálculo, temos 16 bits para rede, 7 para sub-rede e 9 para host. Como tanto os bits de rede e sub-rede são representados pelo algarismo "1" (em binário) e na prática representam a mesma coisa, poderíamos dizer também que temos 23 bits para rede/sub-rede e 9 para hosts. A máscara fica então assim:

11111111.1111111.11111110.00000000

* 1 = rede/sub-rede , 0 = host


      Convertendo para decimal, temos:

255.255.254.0


      Lembrando que quando convertemos a máscara de binário para decimal, "esquecemos" a divisão rede/sub-rede - host e analisamos apenas a divisão por octetos.

      Agora vamos definir qual é o endereço da última sub-rede utilizável. Como não alteramos os dois primeiros octetos, que representam a rede, não precisamos analisá-lo. Vamos analisar apenas os dois últimos octetos. Para descobrirmos qual é a décima sub-rede, vamos colocar o equivalente em binário ao valor 10 em decimal:

10(10) = 0001010(2)


     Para facilitar, como o campo host possui 7 bits, acrescentamos três algarismos binários 0 à esquerda (0001010). Lembrando que acrescentando zeros à esquerda, não alteramos o valor, ou seja, 0001010 = 1010. Agora vamos aplicar o valor descoberto, preenchendo todo o campo host com o algarismo 0:

00010100.00000000

7 bits de sub-rede, 9 bits de host


      Dois detalhes muito importantes:
  • O campo host está preenchido todo com o algarismo 0. É assim que ele deve estar quando calculamos um endereço de sub-rede
  • A décima sub-rede tem o valor 0001010 em binário, que é igual a 10 em decimal;

      Agora que descobrimos o valor em binário, vamos converter para decimal. Um detalhe muito importante é que devemos montá-lo agrupando em octetos, independente do bit representar sub-rede ou host. Veja:

3º octeto: 00010100(2) = 20(10)
4º octeto: 00000000(2) = 0(10)


      Então vamos escrever o endereço completo, "anexando" os dois octetos que calculamos aos dois primeiros octetos:

140.168.20.0


      Agora devemos calcular o endereço de broadcast da última sub-rede utilizável. Voltaremos para o número em binário e iremos manter o valor dos bits que representam a sub-rede, trocando apenas o valor dos bits do campo host de 0 para 1, pois para termos um broadcast dentro da sub-rede, todos os bits do campo host devem ser preenchidos com o valor 1. Veja:

00010101.11111111

      7 bits de sub-rede, 9 bits de host - todos preenchidos com 1 (broadcast)


      Descobrimos o valor em binário, convertendo para decimal, temos:

3º octeto: 00010101(2) = 21(10)
4º octeto: 11111111(2) = 255(10)


      Então vamos escrever o endereço completo, "anexando" o octeto que calculamos aos três primeiros octetos:

140.168.21.255


      Ok..., agora falta apenas classificar o endereço 140.168.155.0 Vamos analisar os dois últimos octeto, começando convertendo os valores deles:

3º octeto: 155(10)= 10011011(2)
4º octeto: 0(10)= 00000000(2)


      Vamos analisar na estrutura sub-rede / host:

10011011.00000000


      Analisando alguns detalhes muito importantes:
  • O campo host não está preenchido todo com o algarismo 0, então este endereço não representa uma sub-rede;
  • O campo host não está preenchido todo com o algarismo 1, então este endereço não representa um broadcast;
  • Não representando uma sub-rede ou um broadcast, este endereço representa um host.

    Para descobrirmos a qual sub-rede pertence este host, podemos utilizar dois métodos:
  1. Colocar o valor 0 em todos os bits de host, sem alterar os bits de rede e sub-rede.
  2. Aplicar uma operação AND do endereço com a máscara de sub-rede

      Com os dois métodos obtemos o mesmo resultado. Eu prefiro o 1º método. Vamos analisar apenas os dois últimos octetos, pois neste caso não alteramos o 1º nem o 2º octetos. Os valores deles são 155 ( 10011011 em binário) e 0(00000000 em binário). O valor dos bits de sub-rede é 1001101 e o valor dos bits do campo host é 1.00000000. Conforme descrevemos anteriormente, vamos alterar o valor dos bits do campo host para 0 e obteremos assim a sub-rede a qual o host pertence. Os octetos em binário ficarão assim:

10011010.00000000


      Agora que descobrimos o valor em binário, vamos converter para decimal. Um detalhe muito importante é que devemos montá-lo agrupando em octetos, independente do bit representar sub-rede ou host. Veja:

3º octeto: 10011010(2) = 154(10)
4º octeto: 00000000(2) = 0(10)


      Concluindo, o host 140.168.155.0, utilizando a máscara 255.255.255.240, pertence a sub-rede 140.168.154.0.

      Vamos demonstrar também como obter o mesmo resultado através de uma operação AND com a máscara de sub-rede. Primeiro vamos passar o endereço 140.168.155.0 e a máscara 255.255.254.0 para binário:

Decimal Binário
140.168.155.0 10001100.10101000.10011011.00000000
255.255.254.0 11111111.11111111.11111110.00000000


      Vamos agora aplicar uma operação AND, algarimo por algarismo. Caso tenha dúvida em como efetuar esta operação, favor visite a seção operações lógicas

10001100.10101000.10011011.00000000
11111111.11111111.11111110.00000000
---------------------------------------------
10001100.10101000.10011010.00000000


      Convertendo 10001100.10101000.10011010.00000000 para decimal, obtemos 140.168.154.0, o mesmo resultado obtido com o outro método.

    Finalizando, vamos escrever os resultados obtidos:
  • Máscara = 255.255.254.0
  • Endereço da décima sub-rede utilizável = 140.168.20.0
  • Endereço de broadcast da décima sub-rede utilizável = 140.168.21.255
  • Classificação do endereço 148.168.155.0 = host
  • A sub-rede a qual pertence o host 148.168.155.0 é 148.168.154.0
