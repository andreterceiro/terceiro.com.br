Resolução dos exercícios : 1 - Primeiro vamos verificar em que classe está o endereço. Convertendo o primeiro octeto para binário, temos:
      192 (10) = 110 00000 (2)

      Descobrimos então que o endereço pertence à classe C, ou seja, podemos utilizar apenas o último octeto para dividir em sub-redes. Para dividir o endereço em 2 sub-redes utilizáveis devmos pegar emprestado 2 bits do campo host, pois 22-2 = 2. Lembrando que subtraimos 2 porque não utilizamos a primeira nem a última sub-rede. Como pegamos 2 bits, o campo host esté menor. Antes ele possuia 8 bits, agora possui 6 (8 - 2 = 6). Temos que verificar se 6 bits atendem nossa necessidade para o némero de hosts, entéo calculamos:
      26-2=62. Lembrando que subtraimos 2 agora porque o primeiro endereço de cada sub-rede representa a prépria sub-rede e o último é utilizado para broadcast dentro da sub-rede. Nossa necessidade é de 60 hosts por sub-rede, entéo com 62 hosts temos nossa necessidade satisfeita.       Vamos entéo calcular a máscara de sub-rede. Ténhamos antes do célculo 24 bits para rede e 8 para host. Apés nosso célculo, temos 24 bits para rede, 2 para sub-rede e 6 para host. Como tanto os bits de rede e sub-rede séo representados pelo algarismo "1" (em binário) e na prética representam a mesma coisa, poderéamos dizer também que temos 26 bits para rede/sub-rede e 6 para hosts. A máscara fica entéo assim:

assim       11111111.1111111.11111111.11000000

      * 1 = rede/sub-rede , 0 = host


      Convertendo para decimal, temos:

      255.255.255.192

      Lembrando que quando convertemos a máscara de binário para decimal, "esquecemos" a diviséo rede/sub-rede - host e analisamos apenas a diviséo por octetos.

      Agora vamos definir qual é o endereço da primeira sub-rede utilizável. Como não alteramos os trés primeiros octetos, que representam a rede, não precisamos analisé-lo. Vamos analisar apenas o último octeto. A primeira sub-rede (chamada normalmente de sub-rede 0) tem como valor no último octeto:

      00000000

      2 bits de sub-rede, 6 bits de host

      Trés detalhes muito importantes:
  • O campo host esté preenchido todo com o algarismo 0. é assim que ele deve estar quando calculamos um endereço de sub-rede
  • A sub-rede 0 tem o valor 00 em binário, que é igual a 0 em decimal;
  • O campo sub-rede esté preenchido com o valor da primeira sub-rede, mas ela não é utilizável (a primeira e a última sub-redes não séo utilizáveis).
      Jé que a primeira sub-rede (sub-rede 0) não é utilizável, vamos calcular o endereço da primeira sub-rede utilizável. Em binário, teremos este valor no último octeto:

      01000000

      2 bits de sub-rede, 6 bits de host

      Analisando novamente alguns detalhes muito importantes:
  • O campo host esté preenchido todo com o algarismo 0. é assim que ele deve estar quando calculamos um endereço de sub-rede
  • A sub-rede 1 tem o valor 01 em binário, que é igual a 1 em decimal;
      Descobrimos o valor em binário, convertendo para decimal, temos:
      01000000(2) = 64(10)

      Entéo vamos escrever o endereço completo, "anexando" o octeto que calculamos aos trés primeiros octetos:

      192.168.0.64

      Agora devemos calcular o endereço de broadcast da 1é sub-rede utilizável. Voltaremos para o némero em binário e iremos manter o valor dos bits que representam a sub-rede, trocando apenas o valor dos bits do campo host de 0 para 1, pois para termos um broadcast dentro da sub-rede, todos os bits do campo host devem ser preenchidos com o valor 1. Veja:       01111111

      2 bits de sub-rede, 6 bits de host - todos preenchidos com 1 (broadcast)

      Descobrimos o valor em binário, convertendo para decimal, temos:
      01111111(2) = 127(10)

      Entéo vamos escrever o endereço completo, "anexando" o octeto que calculamos aos trés primeiros octetos:

      192.168.0.127

Ok..., agora falta apenas classificar o endereço 192.168.0.140. Vamos analisar o último octeto, comeéando convertendo o valor dele:

      140(10) = 10001100(2)

      Vamos analisar na estrutura sub-rede / host:
      10001100

      Analisando alguns detalhes muito importantes:
  • O campo host não esté preenchido todo com o algarismo 0, entéo este endereço não representa uma sub-rede.
  • O campo host não esté preenchido todo com o algarismo 1, entéo este endereço não representa um broadcast.
  • não sendo uma sub-rede ou um broadcast, ele é um endereço de host
      Para descobrirmos a qual sub-rede pertence este host, podemos utilizar dois métodos:
  1. Colocar o valor 0 em todos os bits de host, sem alterar os bits de rede e sub-rede.
  2. Aplicar uma operação AND do endereço com a máscara de sub-rede


      Com os dois métodos obtemos o mesmo resultado. Eu prefiro o 1é método. Vamos analisar apenas o último octeto, pois neste caso não alteramos o 1é, o 2é nem o 3é octetos. O valor dele é 10001100 (140 em binário). O valor dos bits de sub-rede é 10 e o valor dos bits do campo host é 001100. Conforme descrevemos anteriormente, vamos alterar o valor dos bits do campo host para 0 e obteremos assim a sub-rede a qual o host pertence. O octeto em binário ficaré assim:
      10000000.

      Convertendo o resultado obtido para decimal e anexando-o aos trés outros octetos, obteremos a resposta:

      10000000(2) = 128(10)

      Concluindo, o host 192.168.0.140, utilizando a máscara 255.255.255.192, pertence a sub-rede 192.168.0.128.

Decimal binário

      Convertendo 11000000.10101000.00000000.10000000 para decimal, obtemos 192.168.0.128, o mesmo resultado obtido com o outro método.

    Finalizando, vamos escrever os resultados obtidos:
  • máscara = 255.255.255.192
  • Endereço da primeira sub-rede utilizável = 192.168.0.64
  • Endereço de broadcast da 1é sub-rede utilizável = 192.168.0.127
  • Classificação do endereço 192.168.0.140 = host
